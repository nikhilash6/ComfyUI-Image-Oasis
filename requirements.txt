# Oasis Suite (Image Oasis v1.5+) -- Python dependencies
#
# Core runtime dependencies (torch, numpy, Pillow, aiohttp, comfy.*,
# folder_paths, PyAV/av) are provided by ComfyUI itself and are NOT listed
# here.
#
# Image Oasis is designed for graceful degradation: with none of the
# packages below installed, the common-case flow still works
# (checkpoint/diffusion model loading, sampling, refiner pass, and
# algorithmic upscaling). Each dep below enables one specific feature; if
# missing, that feature errors clearly at use-time -- the node still loads.

# Model-based upscaling for the Upscale section's "Model" mode.
# Algorithmic upscaling (lanczos / bicubic / bilinear / nearest-exact / area)
# does NOT require these.
spandrel
spandrel-extra-arches

# Prompt enhancer ("magic wand") -- local GGUF LLM that expands short prompts
# (Image Oasis and LTX2.3 Oasis). NOT enabled by default: the CUDA-accelerated
# install is toolkit-specific (cu121 / cu122 / cu124 / etc.) and a plain
# `pip install` lands the CPU-only wheel, which is much slower. See
# image_oasis/README.md "Prompt enhancer setup" for the install command
# matching your hardware.
# llama-cpp-python

# ---------------------------------------------------------------------------
# Non-pip dependencies (ComfyUI custom nodes -- install via Manager / git).
# None of these are installable from this requirements file.
# ---------------------------------------------------------------------------
#
#   ComfyUI-GGUF
#     Required for the GGUF source type on Image Oasis and LTX2.3 Oasis.
#     Checkpoint / diffusion (safetensors) sources work without it.
#
#   ComfyUI-KJNodes
#     Optional for LTX2.3 Oasis. Needed for LTX2 NAG (Normalized Attention
#     Guidance) so negative prompts work when CFG is 1 (distilled LTX).
#     Without it, distilled runs with an empty negative still work; a
#     non-empty negative at CFG 1 will error until KJNodes is installed.
#
# See the root README "Install" section and ltx23_oasis/README.md.
